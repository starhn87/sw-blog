## 블로그를 새로 만들었어요

안녕하세요.
이승우라고 해요.

기존에 velog.io에서 글을 쓰고 있었는데, 직접 블로그를 만들어보고 싶다는 생각이 들었어요.
그래서 이번에 제로베이스에서 새로운 블로그를 구축하게 되었어요.

### 어떤 기술을 사용했나요?

이 블로그는 아래 기술 스택으로 만들어져 있어요:

- **Next.js 15** — App Router 기반으로 구성했어요
- **MDX** — 마크다운에 React 컴포넌트를 섞어 쓸 수 있어서 좋아요
- **Tailwind CSS** — 유틸리티 기반 스타일링이라 빠르게 작업할 수 있었어요
- **Framer Motion** — 인터랙티브한 애니메이션을 넣고 싶었어요
- **Cloudflare D1** — 조회수, 좋아요, 댓글을 Edge에서 처리해요
- **Claude API** — RAG 기반 AI 챗봇을 달았어요

### 핵심 코드 살펴보기

블로그 핵심 기능 몇 가지를 코드로 소개할게요.

#### MDX 파이프라인

모든 글은 로컬 `.mdx` 파일로 관리돼요. 빌드 타임에 파싱해서 정적 생성하는 구조예요.

```tsx
export async function getPostBySlug(slug: string) {
  const filePath = path.join(POSTS_DIR, `${slug}.mdx`);
  const raw = fs.readFileSync(filePath, "utf-8");
  const { data, content } = matter(raw);

  const mdxSource = await compileMDX({
    source: content,
    options: {
      mdxOptions: {
        rehypePlugins: [
          rehypeSlug,
          [rehypePrettyCode, {
            theme: { dark: "github-dark", light: "github-light" },
          }],
        ],
        remarkPlugins: [remarkGfm],
      },
    },
  });

  return {
    meta: { ...data, slug, readingTime: readingTime(content).text },
    content: mdxSource,
  };
}
```

`rehype-pretty-code`로 서버 사이드에서 코드 하이라이팅을 처리하기 때문에, 클라이언트에 보내는 JS가 거의 없어요. `react-syntax-highlighter` 대비 약 200KB를 절감할 수 있었어요.

#### AI 챗봇 — SSE 스트리밍

챗봇은 Claude API를 SSE(Server-Sent Events)로 스트리밍해요. 빌드 타임에 MDX 글을 청크로 쪼개서 키워드 기반 RAG를 구현했어요.

```tsx
export async function POST(request: Request) {
  const { messages } = (await request.json()) as { messages: Message[] };
  const lastMessage = messages[messages.length - 1].content;

  // 키워드 기반 RAG — 벡터 DB 없이도 충분해요
  const chunks = await findRelevantChunks(lastMessage);
  const context = chunks.map((c) => c.content).join("\n\n");

  const stream = await anthropic.messages.stream({
    model: "claude-haiku-4-5-20251001",
    max_tokens: 1024,
    system: `블로그 콘텐츠를 기반으로 답변하세요.\n\n${context}`,
    messages,
  });

  // SSE 형식으로 클라이언트에 전달
  return new Response(
    new ReadableStream({
      async start(controller) {
        for await (const event of stream) {
          if (event.type === "content_block_delta") {
            const text = event.delta.text;
            controller.enqueue(`data: ${JSON.stringify(text)}\n\n`);
          }
        }
        controller.enqueue("data: [DONE]\n\n");
        controller.close();
      },
    }),
    { headers: { "Content-Type": "text/event-stream" } },
  );
}
```

개인 블로그 규모에서는 벡터 DB 대신 키워드 매칭으로도 충분하더라고요. 외부 의존성 없이 빌드 타임 JSON만으로 동작해요.

#### Cloudflare D1 — Edge에서 좋아요

좋아요는 Cloudflare D1(SQLite)로 Edge에서 바로 처리해요. IP 기반으로 중복을 방지하고, toggle 방식이에요.

```tsx
export async function POST(request: Request) {
  const { slug } = (await request.json()) as { slug: string };
  const visitorId = request.headers.get("cf-connecting-ip") ?? "unknown";
  const db = getDB(getRequestContext().env.DB);

  const existing = await db.select().from(likes)
    .where(and(eq(likes.slug, slug), eq(likes.visitorId, visitorId)));

  if (existing.length > 0) {
    await db.delete(likes)
      .where(and(eq(likes.slug, slug), eq(likes.visitorId, visitorId)));
  } else {
    await db.insert(likes).values({ slug, visitorId });
  }

  const result = await db.select({ count: count() }).from(likes)
    .where(eq(likes.slug, slug));

  return Response.json({
    count: result[0].count,
    liked: existing.length === 0,
  });
}
```
